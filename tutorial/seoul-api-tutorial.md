# 서울 공공 API로 학교 정보 서비스 만들기
### 입문자를 위한 단계별 튜토리얼

---

## 이 튜토리얼로 배우는 것

- 서울 열린데이터광장에서 무료 API 키 발급하는 방법
- API 주소(URL) 구조를 읽는 방법
- API 키를 안전하게 보관하는 방법 (서버 프록시 패턴)
- 받아온 데이터를 차트와 테이블로 보여주는 방법
- Gemini AI와 연결해서 한국어로 데이터를 탐색하는 방법
- Vercel로 인터넷에 공개하는 방법

> 💡 **전제 조건:** Node.js 설치, GitHub 계정, 브라우저 사용 가능 — 코딩 경험이 없어도 따라올 수 있습니다.

## STEP 01 — 공공 API가 뭔가요?

API(Application Programming Interface)는 프로그램끼리 데이터를 주고받는 창구입니다.

**편의점 비유로 이해하기**

- 편의점(서울시 서버)에 물건(학교 데이터)이 있습니다.
- 계산대(API)를 통해서만 물건을 받아갈 수 있습니다.
- 멤버십 카드(API 키)가 있어야 물건을 살 수 있습니다.

**서울 열린데이터광장이란?**

서울시가 운영하는 공공 데이터 포털로, 교통·환경·교육·복지 등 2,000개 이상의 데이터셋을 무료로 제공합니다. 우리가 사용할 `neisSchoolInfoJS`는 서울시 학교 610개의 정보를 담고 있습니다.

---

## STEP 02 — API 키 발급받기

API 키는 "나는 허가받은 사용자입니다"를 증명하는 긴 문자열입니다. 무료로 발급받을 수 있습니다.

**발급 순서**

1. 브라우저에서 `data.seoul.go.kr` 접속
2. 우측 상단 **회원가입** → 이메일 또는 카카오/네이버로 가입
3. 로그인 후 → **마이페이지** → **인증키 발급/관리**
4. **인증키 신청** 버튼 클릭
5. 발급된 키(예: `7559776558746f74...`)를 복사해서 안전한 곳에 저장

> ⚠️ **주의:** 발급받은 키는 절대 타인에게 공유하거나 GitHub에 올리지 마세요. 남이 내 명의로 API를 무단으로 사용할 수 있습니다.

## STEP 03 — API 주소(URL) 구조 이해하기

서울 학교 정보 API의 호출 주소 형식은 다음과 같습니다.

```
http://openapi.seoul.go.kr:8088/{API키}/json/neisSchoolInfoJS/{시작번호}/{끝번호}/{학교명}
```

**각 부분의 의미**

| 부분 | 설명 | 예시 |
|------|------|------|
| `{API키}` | 발급받은 인증키 | `7559776558746f74...` |
| `json` | 응답 형식 (json 또는 xml) | `json` |
| `neisSchoolInfoJS` | 데이터셋 이름 (고정) | — |
| `{시작번호}` | 몇 번째 데이터부터 가져올지 | `1` |
| `{끝번호}` | 몇 번째 데이터까지 가져올지 | `610` |
| `{학교명}` | 검색할 학교명 (전체 조회는 `%20`) | `%20` |

**실제 사용 예시**

```
# 전체 학교 610개 가져오기
http://openapi.seoul.go.kr:8088/내API키/json/neisSchoolInfoJS/1/610/%20

# 1~10번째 학교만 가져오기
http://openapi.seoul.go.kr:8088/내API키/json/neisSchoolInfoJS/1/10/%20

# "광진"이 들어간 학교 검색
http://openapi.seoul.go.kr:8088/내API키/json/neisSchoolInfoJS/1/10/광진
```

## STEP 04 — 응답 데이터 구조 읽기

API를 호출하면 다음과 같은 JSON 형식의 데이터를 돌려줍니다.

```json
{
  "neisSchoolInfoJS": {
    "list_total_count": 610,
    "RESULT": {
      "CODE": "INFO-000",
      "MESSAGE": "정상 처리되었습니다."
    },
    "row": [
      {
        "SCHL_NM": "광진초등학교",
        "SCHL_KND_NM": "초등학교",
        "FNDN_SE": "공립",
        "CMPTNC_OGNZ_NM": "서울강동송파교육지원청",
        "ROAD_NM_ADDR": "서울특별시 광진구 ...",
        "TELNO": "02-450-1234",
        "HMPG_ADDR": "http://gwangjin.es.kr",
        "FNDN_YMD": "19461001"
      },
      ...
    ]
  }
}
```

**주요 필드 설명**

| 필드명 | 의미 |
|--------|------|
| `list_total_count` | 전체 데이터 건수 (610) |
| `RESULT.CODE` | `INFO-000`이면 정상, 다른 코드면 에러 |
| `SCHL_NM` | 학교명 |
| `SCHL_KND_NM` | 학교종류 (초등학교/중학교/고등학교 등) |
| `FNDN_SE` | 설립구분 (공립/사립) |
| `CMPTNC_OGNZ_NM` | 관할 교육지원청 |
| `ROAD_NM_ADDR` | 도로명주소 |
| `FNDN_YMD` | 설립일자 (YYYYMMDD 형식) |
| `HMPG_ADDR` | 홈페이지 URL |

> ✅ **핵심 체크:** 응답을 받으면 항상 `RESULT.CODE === "INFO-000"` 인지 먼저 확인하세요. 다른 코드면 에러 처리를 해야 합니다.

## STEP 05 — API 키 보안: 서버 프록시 패턴

서울 API는 `http://` + 포트 `8088`을 사용합니다. 브라우저에서 직접 이 주소를 호출하면 두 가지 문제가 생깁니다.

1. **Mixed Content 오류:** HTTPS 사이트에서 HTTP API를 직접 호출할 수 없습니다.
2. **API 키 노출:** 브라우저 개발자 도구에서 누구나 키를 볼 수 있습니다.

**해결책: 서버 프록시**

```
사용자 브라우저  →  내 서버(Next.js API Route)  →  서울 공공 API
   (키 없음)         (키 보관, 대신 호출)           (데이터 제공)
```

**코드로 보면 (`app/api/seoul/route.ts`)**

```typescript
// 서버에서만 실행되는 코드
export async function GET() {
  // 환경변수에서 키 읽기 (클라이언트에 절대 노출 안 됨)
  const apiKey = process.env.SEOUL_API_KEY

  // 서버가 직접 서울 API 호출
  const url = `http://openapi.seoul.go.kr:8088/${apiKey}/json/neisSchoolInfoJS/1/610/%20`
  const res = await fetch(url)
  const data = await res.json()

  // 클라이언트에게는 데이터만 전달 (키는 절대 포함 안 됨)
  return NextResponse.json({ rows: data.neisSchoolInfoJS.row })
}
```

**환경변수 설정 (`.env.local` 파일)**

```env
# 이 파일은 .gitignore에 의해 GitHub에 올라가지 않습니다
SEOUL_API_KEY=여기에_발급받은_키_입력
```

## STEP 07 — Gemini AI와 데이터 연결하기

**Gemini API 키는 어디서 받나요?**

Google AI Studio(aistudio.google.com)에서 무료로 발급받을 수 있습니다.

**Gemini 키 저장 방식 (보안)**

서울 API 키와 달리, Gemini 키는 사용자가 직접 입력하고 브라우저 `localStorage`에 저장됩니다. API 호출 시 서버에 `X-Gemini-Key` 헤더로 전달합니다.

```
사용자가 /settings에서 키 입력
        ↓
localStorage에 저장 (브라우저에만)
        ↓
AI 채팅 요청 시 헤더로 서버 전달
        ↓
서버에서 Gemini API 호출
```

> ⚠️ **공용 PC 주의:** 공용 컴퓨터에서 Gemini 키를 입력한 경우, 반드시 `/settings` 페이지에서 **키 삭제** 버튼을 눌러 삭제하세요.

**AI 정확도를 높이는 핵심 기법: 사전 계산된 통계 전달**

원시 데이터(610개 학교의 JSON)를 그대로 AI에게 넘기면 두 가지 문제가 있습니다.

1. 데이터가 너무 커서 AI가 처리하지 못할 수 있습니다 (토큰 한도 초과)
2. AI가 직접 숫자를 계산하면서 오류가 생길 수 있습니다

**해결책: 서버에서 미리 집계 후 전달**

```
서버에서 계산한 통계 예시:
━━━━━━━━━━━━━━━━━━━━━━━
총 학교 수: 610개교
[학교종류별]  초등학교: 304개교 / 중학교: 128개교 / 고등학교: 168개교
[설립구분별]  공립: 486개교 / 사립: 124개교
[가장 오래된 학교 TOP 5]
  경기고등학교(1900년), 배재고등학교(1885년), ...
[연대별 설립 현황]
  1900년대: 12개교 / 1950년대: 89개교 / 1970년대: 201개교 ...
[전체 학교명 목록]
  광진초등학교(초등학교, 공립) / 경기고등학교(고등학교, 사립) ...
━━━━━━━━━━━━━━━━━━━━━━━
```

이렇게 전달하면 AI가 "공립 학교는 몇 개인가요?"라는 질문에 직접 계산 없이 정확하게 486개라고 답할 수 있습니다.

## STEP 09 — Vercel로 인터넷에 공개하기

**Vercel이란?**

Next.js를 만든 회사가 운영하는 무료 배포 플랫폼입니다. GitHub 코드를 연결하면 자동으로 인터넷에 배포해줍니다.

**배포 순서**

1. `vercel.com` 접속 → GitHub 계정으로 로그인
2. **Add New Project** → `philosophyAIEDU/260606seoul` 선택
3. 배포 전 **Environment Variables** 섹션에 아래 내용 추가:

   | Name | Value |
   |------|-------|
   | `SEOUL_API_KEY` | 발급받은 서울 API 키 |

4. **Deploy** 버튼 클릭
5. 배포 완료 후 `yourproject.vercel.app` 주소로 접속 가능

> 💡 **자동 배포:** 이후 GitHub에 코드를 올리면(push) Vercel이 자동으로 새 버전을 배포합니다.

> ⚠️ **중요:** `SEOUL_API_KEY`는 반드시 Vercel 환경변수 설정창에 입력해야 합니다. 코드에 직접 쓰면 GitHub에 공개되어 키가 노출됩니다.

## 이 튜토리얼에서 배운 핵심 개념 정리

| 개념 | 설명 |
|------|------|
| **공공 API** | 정부/공공기관이 제공하는 무료 데이터 창구 |

서울 열린데이터광장에는 이 외에도 다양한 데이터가 있습니다.
